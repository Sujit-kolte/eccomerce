# Razorpay Integration Setup Guide

## Overview

This e-commerce platform uses Razorpay as the payment gateway. Razorpay is a leading payment platform in India that supports credit/debit cards, UPI, netbanking, and more.

## Prerequisites

- Node.js and npm installed
- MongoDB database setup
- Razorpay account (free to create at https://dashboard.razorpay.com)

## Step 1: Create a Razorpay Account

1. Visit [Razorpay Dashboard](https://dashboard.razorpay.com)
2. Sign up for a free account using your email
3. Complete email verification
4. Fill in your business details

## Step 2: Get API Keys

1. Log in to Razorpay Dashboard
2. Go to **Settings** → **API Keys**
3. Copy your:
   - **Key ID** (public key)
   - **Key Secret** (private/secret key)
4. Keep these keys safe - never share the Key Secret

## Step 3: Configure Environment Variables

Create a `.env` file in the server directory with:

```env
# MongoDB
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/dbname

# JWT
JWT_SECRET=your_jwt_secret_key_here
JWT_EXPIRE=7d

# Razorpay Keys
RAZORPAY_KEY_ID=rzp_test_your_key_id_here
RAZORPAY_KEY_SECRET=rzp_test_your_key_secret_here
RAZORPAY_WEBHOOK_SECRET=your_webhook_secret_here

# Server
NODE_ENV=development
PORT=5000
```

## Step 4: Setup Webhook

The webhook allows Razorpay to notify your server of payment status updates.

### For Local Development (using ngrok):

1. **Install ngrok:**

   ```bash
   # Download from https://ngrok.com/download
   # Or install via npm
   npm install -g ngrok
   ```

2. **Start your server:**

   ```bash
   npm run dev
   ```

3. **In a new terminal, start ngrok:**

   ```bash
   ngrok http 5000
   ```

   This will give you a URL like: `https://abc123def456.ngrok.io`

4. **Add Webhook in Razorpay Dashboard:**
   - Go to **Settings** → **Webhooks**
   - Click **Add New Webhook**
   - Webhook URL: `https://abc123def456.ngrok.io/api/payments/webhook`
   - Select events:
     - `payment.authorized`
     - `payment.captured`
     - `payment.failed`
   - Copy the **Webhook Secret** generated by Razorpay
   - Add it to your `.env` file as `RAZORPAY_WEBHOOK_SECRET`

### For Production Deployment:

1. **Deploy your backend** to a hosting service (Heroku, AWS, etc.)
2. **Get the live URL** of your server (e.g., `https://yourdomain.com`)
3. **Add Webhook in Razorpay Dashboard:**
   - Go to **Settings** → **Webhooks**
   - Webhook URL: `https://yourdomain.com/api/payments/webhook`
   - Select the same events
   - Copy and save the **Webhook Secret** to your production `.env`

## Step 5: Update Frontend Configuration

Create `.env` file in the client directory:

```env
REACT_APP_API_URL=http://localhost:5000/api
REACT_APP_RAZORPAY_KEY_ID=rzp_test_your_key_id_here
```

## Step 6: Install Dependencies

```bash

npm install

# Server dependencies
cd server
npm install

# Client dependencies
cd ../client
npm install
```

## Step 7: Start the Application

```bash
# From root directory
npm run dev
```

This will start:

- Backend on: `http://localhost:5000`
- Frontend on: `http://localhost:3000`

## Payment Flow Architecture

```
Client (React)
    ↓ (1) Create Order
Backend (Node/Express)
    ↓ (2) Create Razorpay Order
Razorpay API
    ↓ (3) Return Order ID
Backend
    ↓ (4) Return Order ID to Client
Client
    ↓ (5) Open Razorpay Checkout Modal
Razorpay Checkout
    ↓ (6) User enters payment details
Razorpay API
    ↓ (7) Process payment
Razorpay Webhook
    ↓ (8) Send payment.authorized/captured/failed event
Backend
    ↓ (9) Update order status in database
    ↓ (10) Return success to user
Client
    ↓ (11) Redirect to order confirmation
Order Confirmation Page
```

## Webhook Events Handled

### 1. payment.authorized

- **Trigger:** Payment is authorized but not yet captured
- **Action:** Mark order as paid
- **Used for:** Recurring payments, deferred captures

### 2. payment.captured

- **Trigger:** Payment is successfully captured
- **Action:** Mark order as paid
- **Used for:** Most transactions

### 3. payment.failed

- **Trigger:** Payment failed
- **Action:** Mark order as unpaid, store failure reason
- **Used for:** Retry logic, customer notification

## Webhook Signature Verification

All webhooks are signed using HMAC-SHA256 with your webhook secret. The signature is sent in the `X-Razorpay-Signature` header.

**Verification Process:**

1. Extract the header: `x-razorpay-signature`
2. Create HMAC-SHA256: `hmac_sha256(webhook_secret, raw_body)`
3. Compare with signature (must match exactly)

This ensures webhook authenticity and prevents unauthorized requests.

## API Endpoints

### Create Payment Intent

```
POST /api/payments/create-payment-intent
Headers: Authorization: Bearer <token>
Body: {
  amount: number,
  orderId: string,
  email: string
}
Response: {
  orderId: string,
  amount: number,
  currency: string
}
```

### Confirm Payment (Client-side verification)

```
POST /api/payments/confirm-payment
Headers: Authorization: Bearer <token>
Body: {
  razorpayPaymentId: string,
  razorpayOrderId: string,
  razorpaySignature: string,
  orderId: string
}
Response: {
  message: string,
  order: Order object
}
```

### Webhook (Called by Razorpay)

```
POST /api/payments/webhook
Headers: X-Razorpay-Signature: <signature>
Body: {
  event: string,
  payload: object
}
```

## Troubleshooting

### Webhook Not Receiving Events

1. Check ngrok tunnel is running
2. Verify webhook URL is correct in Razorpay dashboard
3. Check server logs for errors
4. Verify RAZORPAY_WEBHOOK_SECRET matches in dashboard

### Signature Verification Failed

1. Ensure RAZORPAY_WEBHOOK_SECRET is correct
2. Check that raw body is being used (not parsed JSON)
3. Verify secret is not accidentally modified

### Test Payment Not Working

1. Ensure test/sandbox keys are being used
2. Check that Key ID and Key Secret are correct
3. Verify account has test mode enabled
4. Check browser console for errors

### Order Not Updating After Payment

1. Verify MongoDB connection is active
2. Check that orderId in payment notes matches database
3. Review server logs for any errors
4. Verify webhook is actually being called (check logs)

## Security Best Practices

1. **Never expose API secrets** - Only use Key Secret on backend
2. **Always verify signatures** - Never trust webhook data without verification
3. **Use HTTPS** - In production, always use SSL/TLS
4. **Rotate webhook secrets** - Periodically regenerate secrets
5. **Validate user input** - Sanitize all payment-related inputs
6. **Rate limit endpoints** - Prevent abuse of payment endpoints
7. **Log transactions** - Keep audit trail of payments
8. **Handle idempotency** - Prevent duplicate charges from repeated requests

## Going Live (Production)

1. Switch from Test to Live keys in Razorpay Dashboard
2. Update `.env` with live Key ID and Key Secret
3. Update webhook URL to your production domain
4. Test with small amounts before full launch
5. Implement proper error handling and user notifications
6. Set up monitoring and alerts for payment failures
7. Consider PCI compliance requirements
8. Enable 2FA on Razorpay account
9. Set up settlement account details
10. Review Razorpay documentation for latest updates
